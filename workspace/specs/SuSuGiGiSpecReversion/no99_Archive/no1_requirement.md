# Requirement

## 手動逐筆記帳讓帳本不可信，使用者最終因維護負擔放棄

**根源性：**
    - 主要使用者：希望掌控財務的自學成年人。
    - JTBD：在不增加認知負荷的前提下，知道錢花到哪裡、確認可用餘額。
    - 現況工具：
        - 手動記帳 App 逐筆輸入
        - 試算表
    - 結構性困境——注意力經濟：為了回答剩多少錢這個問題，使用者必須先成為資料輸入員。手動記帳違反人性，且一旦漏記或記錯，整個帳本的餘額就失去參考價值。使用者付出大量勞力，卻只得到一個不可信的數字，最終導致放棄。
    - Data vs Information 落差：銀行只能告訴使用者錢去了哪裡 Where，只有記帳 App 能幫使用者定義錢是為了什麼而花 Why。一筆家樂福 2,500 元的消費，可能包含幫公司代墊的文具、小孩的奶粉、家用食材——這類拆帳與代墊款的語意拆解，是金流 App 永遠無法自動解決的痛點，因為它涉及人的社交關係與商業邏輯。
    - 競品差距：市面所有主流記帳 App 均要求使用者擔任資料輸入員，沒有解決輸入負擔本身。
**取代性：**
    - 選用解法：AI 審閱模式——將使用者從資料輸入員轉型為決策者。
        - 資料整合：AI Agent 自動讀取各資料來源，包含銀行明細、發票載具、電子支付紀錄，產出初步的消費與記帳紀錄。
        - 確認審閱：主要互動模式從逐筆填空輸入轉變為審核驗證，操作方式與 Code Review 相同——確認 AI Agent 產出的帳務紀錄是否正確。
        - 非破壞性修正：發現 AI 分類錯誤時，透過對話修正 AI 的分類規則，而非手動修改單筆資料。規則修正後，系統自動套用新規則並重算所有歷史數據。
    - 向外取代：審閱操作比逐筆輸入的切換阻力低，首次設定完成後日常互動量極小。
    - 向內被取代：AI 自動記帳本身就是此解法的核心技術，不構成被取代風險。
    - 真正的風險是 AI 分類錯誤率過高，導致使用者回歸不信任狀態。
**效益性：**
    - 留存率是付費轉化的前置條件。
    - Tier 0 免費版透過低門檻建立使用習慣。
    - Tier 1 雲端同步成為有習慣後的自然升級動機。
    - Lifecycle Risk：AI 模型能力持續提升使自動分類準確率有改善空間，ROI 曲線向上，無需折現。
**含金量結論：**
    - 高。整個產品飛輪的起點，Accounting App 的核心存在前提。納入整合層。

## 銀行與金流 App 的利益方向與使用者財務健康目標相反，無法提供中立建議

**根源性：**
    - 主要使用者：對個人財務有主動管理意識的成年人，不信任銀行 App 提供的理財建議。
    - JTBD：需要一個不受銀行或金融機構利益影響的財務管理 Agent。
    - 現況工具：銀行 App 的帳目查詢功能。
    - 競品差距：銀行 App 介面設計導向促進消費、貸款與分期，與記帳 App 的財務自律目標根本對立。
    - 使用者不會請賭場老闆幫忙管錢包。
    - 隱私與資料主權風險：將所有財務資料集中於單一超級 App，等同讓平台擁有上帝視角，可能根據消費能力進行差別定價，即殺熟。使用者需要處理極度私密的財務邏輯與標籤，例如私房錢——這類資料連開發者都不應該看到。
**取代性：**
    - 選用解法：Local-First 獨立記帳 App，財務資料儲存於使用者裝置，不依賴任何金融機構。
    - 向外取代：主打隱私訴求與利益中立性，對已有隱私意識的使用者切換阻力低。
    - 向內被取代：超級 App 整合趨勢短期內無法解決利益衝突的根本問題。
    - Local-First 架構在隱私法規日趨嚴格的市場是護城河，而非技術包袱。隱私法規趨嚴的全球趨勢持續強化此優勢，使 Local-First 的防禦性隨時間增長。
**效益性：**
    - 難以短期量化，但作為差異化定位的長期護城河，間接支撐付費轉化率。
    - Lifecycle Risk：隱私法規趨嚴使此優勢在長期有強化空間，ROI 折現率低。
**含金量結論：**
    - 中。差異化定位，非直接轉化驅動，但影響品牌定位決策。納入整合層作為設計約束。

## 資產碎片化讓使用者無法掌握跨金融孤島的全局財務視角

**根源性：**
    - 主要使用者：生活財務結構複雜的成年人。
        - 多個帳戶
        - 跨幣別支出
        - 社交代墊款
    - 無法被單一平台統整的資產類型持續擴張：
        - 現金——雖然減少但不會消失
        - 人情債——午餐先付、朋友欠款，這類社交債務無數位軌跡，只存在於記憶或記帳 App
        - 跨國資產——同時擁有台灣、美國、日本的金融帳戶，無單一超級 App 能跨境整合所有 API
        - Crypto 與冷錢包——去中心化資產本質上抗拒被單一中心化 App 收編
        - 隱形資產——航空哩程、信用卡紅利點數
    - 生活階段的必然升級：使用者開始經營副業、有了家庭帳、持有多個投資帳戶後，財務結構的複雜度不可逆地上升。市面 App 太簡單，無法適應使用者千變萬化的人生。
    - JTBD：取得超越單一銀行帳戶視角的全局可用資產數字。
    - 現況工具：多個 App 分開記錄加試算表整合，或直接放棄整合。
    - 競品差距：市面主流記帳 App 不支援代墊款邏輯與跨幣別轉帳的拆帳計算，這類社交財務邏輯是銀行 API 永遠無法自動解決的問題。
**取代性：**
    - 選用解法：邏輯引擎加規則中心，讓使用者定義每一筆錢的意義，並以非破壞性修正重新套用歷史資料。
    - App 作為 Meta-Layer 元數據層：縫合各個無法互通的金融孤島，提供上帝視角的資產總覽。App 不試圖取代任何金融平台，而是在所有平台之上提供解讀與整合。
    - 向外取代：取代試算表手動整合，切換阻力在於初次規則設定的學習曲線，透過填空式規則介面降低門檻。
    - 向內被取代：AI 能輔助規則建議，但規則本身反映使用者獨特的財務結構，AI 無法完全取代主觀意圖定義。
    - 此解法隨 AI 能力提升而加強，不被取代。
**效益性：**
    - 邏輯引擎是 Tier 2 Pro 方案的核心價值支撐，定價 90 美元每月，ARPU 提升的主要槓桿。
    - 對應有複雜財務需求的高付費意願族群，留存率高。
    - Lifecycle Risk：此需求因使用者財務結構複雜化而增長，ROI 長期向上。
**含金量結論：**
    - 高。ARPU 提升的核心槓桿，Tier 2 與 Tier 3 商業基礎。納入整合層。

## 使用者缺乏可信的前瞻性財務決策依據，現有工具只能回顧無法預測

**根源性：**
    - 主要使用者：已建立記帳習慣、累積一定歷史數據的成年人，正面臨重大財務決策。
    - JTBD：在消費當下或人生規劃時刻，取得基於個人真實數據的前瞻性財務判斷依據。
    - 核心場景——即時償債能力：我想買這台車，但我目前的財務狀況允許嗎？既然固定支出無法改變，這個月真正還剩多少閒錢可以花？
    - 核心場景——長期人生規劃：我想結婚、買房，依我現在的存錢速度，幾年後可以達成？我在同齡人中算是有錢嗎？
    - 現況工具：
        - 直覺判斷與心算
        - 試算表自行建模推演
        - 付費人類財務顧問
    - 競品差距：所有記帳 App 只提供歷史報表——圓餅圖、趨勢圖——告訴使用者過去花了多少，但沒有任何一個能回答未來會怎樣。人類財務顧問價格高且無法即時回應日常消費決策。
**取代性：**
    - 選用解法：AI Advisor——現金流預測模型、財務健康評分系統、自然語言問答介面。
    - 此解法的前提是使用者已在 Tier 0 到 Tier 2 累積了結構化、經規則校正的高品質財務數據。沒有可信的歷史數據，任何預測都無意義。
    - 向外取代：比人類財務顧問更即時、更便宜、更個人化。比直覺判斷更有數據基礎。比試算表建模更易用且自動更新。
    - 向內被取代：通用 AI 助手能提供泛用財務建議，但缺乏與個人結構化財務數據的深度整合。本產品的護城河在於 AI 直接存取使用者經 Logic Engine 校正後的完整帳務，這是外部 AI 無法取得的私有數據。
**效益性：**
    - Tier 3 Intelligence 方案定價 140 美元每月，是使用者終身價值最大化的頂層。
    - 異常偵測與主動通知提升黏著度，降低流失率。
    - 啟動條件嚴格：必須等前置 Tier 的使用者基數成熟，才有足夠的數據密度支撐預測品質。
    - Lifecycle Risk：AI 能力持續提升使預測準確度有改善空間，ROI 曲線隨技術進步向上。但短期內使用者基數未達門檻前，此模組的投入產出比偏低。
**含金量結論：**
    - 中高。ARPU 天花板與使用者終身價值的最大化槓桿，但啟動時間依賴前置 Tier 成熟度。納入整合層，排序在 Accounting App 與 Logic Engine 之後。

## B端市場缺乏真實且即時的微觀消費行為數據，問卷與信用卡數據皆有結構性盲區

**根源性：**
    - 主要使用者：需要真實消費洞察的 B端機構——市場研究員、品牌商、金融風控部門、學術研究者。
    - JTBD：取得比問卷更真實、比信用卡數據更完整、且帶有消費意圖維度的即時行為數據。
    - 現況工具：
        - 問卷調查——不準確且延遲，受訪者回憶偏差與社會期望偏差嚴重
        - 信用卡消費分析——只有 Where，沒有 Why，且覆蓋範圍受限於特定支付管道
        - 市調公司報告——昂貴、更新頻率低、粒度粗
    - 競品差距：沒有任何現有數據來源能同時提供交易事實與消費意圖。銀行知道使用者在哪裡花了多少錢，但不知道那筆錢是代墊、自用、還是送禮。只有使用者主動定義過交易意義的記帳系統，才擁有這個 Why 維度。
**取代性：**
    - 選用解法：去識別化聚合消費數據服務，透過市場情資儀表板與總經數據 API 服務 B端。
    - 嚴格執行 K-Anonymity 去識別化處理，絕無個人可識別資訊外流。隱私絕對優先。
    - Logic Engine 是護城河：使用者透過規則中心主動定義每筆交易的意義，這些語意標注是獨有的數據資產，無法從原始銀行交易紀錄中推導。
    - 向外取代：提供由無數微觀真實交易匯聚而成的另類數據 Alternative Data，反映最即時的經濟脈動，比問卷更真實、比信用卡數據更完整。
    - 向內被取代：Open Banking APIs 能提供原始交易數據，但缺乏意圖維度。此解法的 Why 數據層是 Open Banking 無法複製的。
**效益性：**
    - Tier B 企業版依專案報價或企業訂閱制，500 美元以上每月。
    - 營收回饋 C端研發形成飛輪——B端收入支持 C端產品迭代，讓使用者能以低成本享受頂級的決策輔助工具。
    - 啟動條件：C端使用者基數必須達到統計顯著的規模門檻，聚合數據才有商業價值。
    - Lifecycle Risk：數據價值隨使用者基數增長而指數級提升，ROI 長期向上，但前期為零收入的純投入階段。
**含金量結論：**
    - 中。長期策略性收入來源，形成 C端與 B端的正向飛輪，但依賴 C端使用者基數達到規模門檻。納入整合層，排序在所有 C端需求之後。

## SharedLedger — 共用帳本

**根源性：**
    - 主要使用者：與固定成員共同生活的成年人，例如家庭或室友群體。
    - JTBD：自動追蹤多人共同支出並計算每人應付金額，省去手動對帳。
    - 現況工具：
        - Splitwise
        - 家庭共用試算表
    - 競品差距：Splitwise 等專屬拆帳工具已高度成熟，市場滲透率高且免費。
    - 使用者習慣門檻：單一使用者養成記帳習慣已屬不易，要求多位使用者同時具備記帳習慣的機率極低。
**取代性：**
    - 選用解法：在 Local-First 架構上疊加多裝置即時同步與多帳本隔離機制。
    - 向外取代：Splitwise 使用者切換阻力極高，拆帳需求不構成個人財務管理的延伸訴求，無差異化空間。
    - 向內被取代：多人即時同步需求與 Local-First 設計原則根本衝突，技術實作代價超過整個個人帳本模組。
    - 架構衝突難解：無論採用共用單本帳本或多本帳本連結方案，都會面臨難以調和的問題——共用單本帳本需要細緻的權限控制以保護隱私，多本帳本連結會產生資料斷頭與同步一致性問題。
    - 產品複雜度失衡：共用帳本需要完整的權限管理系統，包含交易可見性控制與成員權限分級，帶來產品設計複雜度大幅提升、開發成本顯著增加、使用者操作學習曲線陡峭。
**效益性：**
    - 付費轉化驅動力低，拆帳族群更傾向使用免費專屬工具而非複合記帳 App。
    - 導入共用帳本使產品定位從個人財務工具轉向社交金融協作，稀釋核心訴求。
**含金量結論：**
    - 低。競品飽和，架構成本高，定位稀釋風險顯著。不納入本產品。
